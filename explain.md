# Andaman & Nicobar Forest Fragmentation Analysis

This document explains the metrics, charts, inference generation, and interactive map implementation used in our Streamlit Dashboard.

---

## 1. Key Metrics Explained

These metrics provide a top-level summary of the selected island or aggregated island group:

- **Island Area (e.g., 1328.6 km²)**: The total physical land area of the island(s), derived natively from the GADM spatial vectors (GeoPackage/Shapefile). For aggregated groups, this is the sum of the areas of all constituent islands.
- **Distance to Mainland (e.g., 1366 km)**: The shortest geographical distance from the center of the island to mainland India. For aggregated groups, this is a weighted mean calculated relative to the area of each island.
- **Latest CRR (e.g., 0.463)**: The Core Retention Ratio (CRR) for the most recent year. Mathematically, it is plotted as `Total Core Area (TCA) / Total Area (TA)`.
  - A value of **1.0** indicates absolute pure interior forest with zero edge disruption.
  - A value near **0.0** means the forest is entirely broken into vulnerable "edges" with no core habitat.
- **Cumulative Loss (e.g., 2.98060%)**: The total percentage of forest area lost since the year 2000. We compute this mathematically from true pixel-level deforestation counts directly layered over the 10m-resolution Hansen GFC raster datasets, providing exact high-precision decimal accuracy.

---

## 2. Dynamic Charts Explained

The dashboard uses matplotlib and Altair to render timeseries charts reflecting fragmentation dynamics spanning from 2000 to 2023.

- **Core Retention Ratio (CRR) Over Time**:
  Tracks the interior habitability of the forest year by year. A critical threshold line is rendered at **`0.5`**. Falling below this red line implies that the remaining forest has become fundamentally edge-dominated—meaning the ecosystem structure is suffering high stress.
- **Fragmentation Index (FI) Over Time**:
  Calculated as `(Edge Density * Patch Density) / Percentage of Landscape`. This composite metric spikes during years where continuous forest covers are abruptly broken and fragmented into disconnected patches.
- **Isolation-Weighted Fragmentation Index (IWFI)**:
  An advanced ecological metric combining internal fragmentation with external isolation. It is formulated as: `FI * ln(ENN_MN + 1) * ln(Distance_to_Mainland + 1)`. It reflects how severe a fragmentation event is given how far the island patch is from other potential biological reservoirs (both locally and mainland).

## 3. How "Quick Inferences" and "Threshold Timelines" are Generated

**Question:** _Are the inferences for all the islands manually given or generated by automatic scripts?_

**Answer:** They are **100% automatically generated** on-the-fly by our Python scripts using Pandas dataframe logic.

There is no manual text entry. The script evaluates the mathematical data for the currently selected island (or group) over the chosen slider year range.

- **Core Stress**: The script checks which years `CRR < 0.5`.
- **Critical Loss**: The script flags the exact year the `Cumulative Loss %` crossed `30%`, if any.
- **FI Shock**: The script dynamically calculates the 90th percentile for the FI values of the selected range. If a year's FI exceeds this top 10% bound, it is flagged as an abrupt "FI shock".
- Based on these automatic Boolean triggers (`0` or `1`), Altair renders the Heatmap/Timeline visually. The **text summaries** underneath are constructed via Python formatted strings combining those dynamic counts (e.g., _f"- First year crossing 30% cumulative loss: {first_cross_year}"_).

## 4. Interactive Map Implementation Process

**Question:** _What is the process used to add the interactive map in the portal?_

We use a combination of local Raster processing (`rasterio`), web mapping (`folium`), and Streamlit integration (`st_folium`).

- **Extract & Cookie-Cut Rasters:** Huge multi-gigabyte `.tif` files (like Hansen and ESA) are processed offline. We use Python to "cookie-cut" the exact geometric shape of the island.
- **Generate Transparent Tiles:** Instead of reading massive `.tif` files live (which would lag the server), we export the strictly bounded forest pixels as lightweight, perfectly transparent `.png` image tiles. This is what you see in `map_tiles`.
- **Build the Bounding Box Map:** When you select an island, `map_component.py` looks up its literal South, North, West, and East GPS coordinates. It initiates a `folium.Map` locked directly onto those bounds to set the perfect zoom and frame.
- **Render in Streamlit:** Finally, the Python script takes the `.png` tile, encodes it as a base64 string, and injects it natively as an `ImageOverlay` mathematically pinned to those map coordinates. The `st_folium` library then pushes this completed, interactive HTML widget directly into the sidebar interface.
